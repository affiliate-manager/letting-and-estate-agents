# Letting & Estate Agent Finder

The UK's most transparent letting & estate agent comparison platform. Find, compare and choose with confidence.

## Overview

A static website that lets users search and compare 999+ UK letting & estate agents by postcode or area. All data is loaded client-side from a static JSON file — no backend required.

## Features

- **Agent Trust Score** — Proprietary 50-100 score based on reviews, regulatory compliance, experience & transparency
- **Fee Comparison** — Transparent fee breakdowns for lettings management, tenant find, sales and more
- **Regulatory Shield** — ARLA, Client Money Protection, Property Ombudsman compliance checks
- **Category Filters** — Filter by Letting, Estate, or Both services
- **Experience Segments** — New, Growing, Established, Veteran agent classification
- **Side-by-Side Compare** — Compare up to 3 agents on all metrics
- **Area Insights** — Local market intelligence by postcode area

## Tech Stack

- **Frontend:** HTML5, CSS3, Vanilla JavaScript (ES6 modules)
- **Data:** Static `data/agents.json` (~1.7MB)
- **Auth:** Firebase Authentication (optional, demo mode by default)
- **Hosting:** Any static file server (Cloudways, Cloudflare Pages, Vercel, Netlify)


## Updating Agent Data

The `data/agents.json` file is generated by a separate scraper pipeline. To update:

1. Run the scraper to generate new `output_with_data.csv`
2. Run `convert_csv.py --output path/to/this/repo/data/agents.json`
3. Commit and push — deployment auto-updates

## Deployment (Cloudways via Git)

1. Connect this GitHub repo in Cloudways Dashboard → Application → Deployment Via Git
2. Set deployment branch to `main`
3. Deploy — all files go to `public_html/`
4. Every `git push` to `main` triggers auto-deployment

## Firebase Setup (Optional)

To enable real authentication (instead of demo mode), update `js/config.js` with your Firebase project credentials. See the [Firebase docs](https://firebase.google.com/docs/web/setup).
